# 自动扳机系统G-Hub驱动升级说明

## 📋 升级概述

自动扳机系统已从Win32 API升级为使用罗技G-Hub驱动，提供更稳定、更安全的硬件级鼠标控制。

## 🔄 主要变更

### 驱动变更
- **之前**: 使用Win32 API (`win32api.mouse_event`)
- **现在**: 使用罗技G-Hub驱动 (`ghub_click`)

### 技术优势
| 特性 | Win32 API | G-Hub驱动 |
|------|-----------|-----------|
| 控制级别 | 软件级 | 硬件级 |
| 延迟 | 较高 | 更低 |
| 稳定性 | 一般 | 更稳定 |
| 兼容性 | 通用 | 罗技鼠标专用 |
| 安全性 | 一般 | 更高 |

## 🛠️ 技术实现

### 代码变更
```python
# 之前的实现
win32api.mouse_event(win32con.MOUSEEVENTF_LEFTDOWN, 0, 0)
time.sleep(0.01)
win32api.mouse_event(win32con.MOUSEEVENTF_LEFTUP, 0, 0)

# 现在的实现
ghub_click()  # 硬件级点击，内置延迟控制
```

### 依赖变更
- **移除**: `win32api`, `win32con`
- **新增**: `mouse_driver.MouseMove.ghub_click`

## ✅ 兼容性

### 支持的设备
- 罗技G系列鼠标
- 安装了G-Hub软件的系统
- 支持G-Hub驱动的罗技设备

### 系统要求
- Windows操作系统
- 已安装罗技G-Hub软件
- 罗技兼容鼠标设备

## 🎯 使用体验

### 改进点
1. **更快响应**: 硬件级控制减少延迟
2. **更稳定**: 直接与硬件通信，减少系统干扰
3. **更安全**: 使用官方驱动接口
4. **统一性**: 与瞄准系统使用相同驱动

### 用户体验
- 射击响应更快
- 连发更稳定
- 与瞄准功能协调性更好
- 整体系统更流畅

## 📝 注意事项

1. **设备要求**: 需要罗技G系列鼠标
2. **软件依赖**: 必须安装G-Hub软件
3. **驱动状态**: 确保G-Hub驱动正常工作
4. **兼容性**: 非罗技鼠标用户需要使用罗技设备

## 🔧 故障排除

### 常见问题
1. **点击无响应**: 检查G-Hub软件是否运行
2. **设备未识别**: 确认鼠标是否为罗技G系列
3. **驱动错误**: 重新安装G-Hub软件

### 解决方案
- 重启G-Hub软件
- 检查设备连接
- 更新G-Hub到最新版本
- 确认鼠标设备兼容性
