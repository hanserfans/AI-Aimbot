# 🎮 G-Hub 鼠标移动测试指南

## 📋 概述

本指南提供了两个专门的测试脚本，用于验证进入游戏后的 G-Hub 鼠标移动功能。

## 🔧 测试脚本说明

### 1. 简化版测试脚本 (`simple_ghub_test.py`)

**特点**:
- ✅ 快速验证基本功能
- ✅ 简单的左右移动模式
- ✅ 实时反馈
- ✅ 安全退出机制

**使用方法**:
```bash
python simple_ghub_test.py
```

**控制方式**:
- **激活**: 按住 `Caps Lock` 键
- **退出**: 按 `Q` 键

### 2. 完整版测试脚本 (`test_ghub_ingame.py`)

**特点**:
- ✅ 自动检测游戏窗口
- ✅ 多种移动模式 (圆形、方形、直线、随机)
- ✅ 自动模式切换 (每5秒)
- ✅ 详细的状态显示

**使用方法**:
```bash
python test_ghub_ingame.py
```

**控制方式**:
- **激活**: 按住 `Caps Lock` 键
- **退出**: 按 `Q` 键


### 执行测试

#### 方案一: 快速测试 (推荐新手)

1. **启动简化版测试**:
   ```bash
   python simple_ghub_test.py
   ```

2. **验证导入**:
   - 看到 "✅ G-Hub 驱动导入成功" 表示正常

3. **开始测试**:
   - 按回车键开始
   - 按住 `Caps Lock` 键观察鼠标移动
   - 应该看到水平左右移动

4. **退出测试**:
   - 按 `Q` 键安全退出

#### 方案二: 完整测试 (推荐进阶用户)

1. **启动完整版测试**:
   ```bash
   python test_ghub_ingame.py
   ```

2. **自动窗口检测**:
   - 脚本会自动查找游戏窗口
   - 支持 VALORANT, CS:GO, Apex Legends 等

3. **多模式测试**:
   - 按住 `Caps Lock` 键激活
   - 观察不同的移动模式:
     - 圆形移动
     - 方形移动  
     - 直线移动
     - 随机移动

4. **退出测试**:
   - 按 `Q` 键安全退出

## 🔍 测试结果判断

### ✅ 成功标志

1. **导入成功**:
   ```
   ✅ G-Hub 驱动导入成功
   ```

2. **移动正常**:
   ```
   🖱️ 已执行 X 次移动 (方向: 右/左)
   🖱️ 已执行 X 次移动 (模式: circle/square/line/random)
   ```

3. **响应及时**:
   - 按下 `Caps Lock` 立即开始移动
   - 松开 `Caps Lock` 立即停止移动

### ❌ 问题排查

#### 1. 导入失败
```
❌ G-Hub 驱动导入失败: No module named 'mouse_driver'
```
**解决方案**:
- 确保在正确的项目目录运行
- 检查 `mouse_driver` 文件夹是否存在

#### 2. 设备未找到
```
Warning: G-Hub device not found
```
**解决方案**:
- 安装并启动 Logitech G-Hub 软件
- 连接 Logitech 设备 (鼠标/键盘/耳机)
- 以管理员权限运行脚本

#### 3. 鼠标不移动
**可能原因**:
- `Caps Lock` 键未正确按下
- 游戏窗口未激活
- 权限不足

**解决方案**:
- 确认 `Caps Lock` 指示灯亮起
- 点击游戏窗口确保其处于前台
- 以管理员身份重新运行

## 📊 性能参数

### 默认配置
```python
MOVEMENT_AMP = 0.4      # 移动幅度
UPDATE_RATE = 20Hz      # 简化版更新频率  
UPDATE_RATE = 100Hz     # 完整版更新频率
```

### 调整建议
- **移动太快**: 降低 `MOVEMENT_AMP` 到 0.2-0.3
- **移动太慢**: 提高 `MOVEMENT_AMP` 到 0.5-0.7
- **需要更平滑**: 提高更新频率

## 📞 技术支持

### 常见问题

**Q: 为什么选择 Caps Lock 而不是右键?**
A: 为了与主程序保持一致，便于测试。实际使用中 G-Hub 模式通常使用右键。

**Q: 可以修改激活键吗?**
A: 可以，修改脚本中的 `0x14` (Caps Lock) 为其他键码。

**Q: 测试脚本会影响游戏吗?**
A: 不会，测试脚本仅移动鼠标，不会修改游戏文件或设置。
