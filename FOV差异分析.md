# FOV差异分析

## 🔍 问题描述

你观察到的FOV数值差异：
- **游戏FOV**: H=103.0°, V=76.3°
- **有效FOV**: H=12.88°, V=15.26°

## 📊 差异原因分析

### 1. 游戏FOV vs 有效FOV的概念区别

**游戏FOV (103.0°, 76.3°)**：
- 这是游戏的**完整视野角度**
- 覆盖整个游戏窗口 (2560x1600)
- 水平FOV: 103.0° (用户设置)
- 垂直FOV: 76.3° (根据宽高比计算得出)

**有效FOV (12.88°, 15.26°)**：
- 这是**检测区域**对应的视野角度
- 只覆盖检测图像 (320x320)
- 是游戏FOV的一个**小部分**

### 2. 计算逻辑

```python
# 捕获区域比例
capture_ratio_h = detection_size / game_width  # 320 / 2560 = 0.125
capture_ratio_v = detection_size / game_height # 320 / 1600 = 0.2

# 有效FOV = 游戏FOV × 捕获比例
effective_fov_h = 103.0 × 0.125 = 12.875° ≈ 12.88°
effective_fov_v = 76.3 × 0.2 = 15.26°
```

### 3. 视觉化解释

```

│                                                         │
```

## ✅ 结论

**这个差异是正常的！**

- 游戏FOV是整个屏幕的视野角度
- 有效FOV是检测区域（屏幕中心320×320像素）的视野角度
- 检测区域只占屏幕的一小部分，所以有效FOV远小于游戏FOV
- 这个设计是合理的，因为自瞄只需要关注屏幕中心区域

## 🎯 实际意义

1. **精确瞄准**：有效FOV小意味着检测精度高
2. **性能优化**：只处理中心区域，提高处理速度
3. **实用性**：大部分目标都会出现在屏幕中心区域

这个FOV差异不是问题，而是系统设计的特性！
